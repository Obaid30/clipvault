A
Create the Storage bucket
1
Open Storage in Supabase
In your Supabase project, click Storage in the left sidebar.
2
Create a new bucket
Click New bucket and fill in:

Name: videos
Public bucket: Toggle ON (so video URLs work)
File size limit: 524288000 (= 500MB in bytes)
Allowed MIME types: video/mp4,video/quicktime,video/webm
3
Click Save
Your videos bucket is created. Videos will be stored here as user_id/timestamp_filename.mp4
Supabase free tier gives you 1GB of storage. That's about 5–10 minutes of video footage — enough to launch and get your first users.
B
Create the videos database table
1
Open the SQL Editor
In Supabase left sidebar, click SQL Editor, then click New query.
2
Paste this SQL and run it
Copy the code below exactly, paste it into the SQL editor, and click Run.
Creates the videos tableSQL
-- Create videos table
CREATE TABLE IF NOT EXISTS videos (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE,
  filename text NOT NULL,
  title text,
  storage_path text,
  video_url text,
  thumbnail_url text,
  size_bytes bigint,
  duration_secs integer,
  status text DEFAULT 'uploading',
  clip_count integer DEFAULT 0,
  created_at timestamptz DEFAULT now()
);

-- Users can only see their own videos
ALTER TABLE videos ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users see own videos"
  ON videos FOR ALL
  USING (auth.uid() = user_id);

-- Users can insert their own videos
CREATE POLICY "Users insert own videos"
  ON videos FOR INSERT
  WITH CHECK (auth.uid() = user_id);
3
Click Run → you should see "Success"
The table is created. Click Table Editor in the sidebar and you'll see videos listed there.
Row Level Security means users can only ever see their own videos — nobody can access someone else's data. This is already set up in the SQL above.
C
Set storage access rules
1
Go to Storage → Policies
In Supabase, click Storage → then Policies at the top. Click New Policy on the videos bucket.
2
Run this SQL in the SQL Editor instead (easier)
Paste and run this in SQL Editor → New query:
Storage access policiesSQL
-- Logged-in users can upload to their own folder
CREATE POLICY "Users upload own videos"
  ON storage.objects FOR INSERT
  TO authenticated
  WITH CHECK (bucket_id = 'videos' AND
    (storage.foldername(name))[1] = auth.uid()::text);

-- Logged-in users can read/delete their own files
CREATE POLICY "Users manage own videos"
  ON storage.objects FOR ALL
  TO authenticated
  USING (bucket_id = 'videos' AND
    (storage.foldername(name))[1] = auth.uid()::text);

-- Anyone can view public video URLs
CREATE POLICY "Public video access"
  ON storage.objects FOR SELECT
  TO anon
  USING (bucket_id = 'videos');
If you see an error saying the policy already exists, that's fine — just ignore it and move on.
