🔧 Supabase Setup Guide

Follow these steps to add real user accounts to ClipVault.
No coding needed — just clicks.

0 of 5 steps complete
1
Create your Supabase account
Free forever — no credit card needed
1
Open Supabase
Go to supabase.com ↗ and click the green Start your project button.
2
Sign up with GitHub
Click Continue with GitHub. If you don't have GitHub, click "Sign up" first at github.com — it's free and takes 1 minute.
3
You're in
You'll land on the Supabase dashboard. This is your control panel for the whole backend.
Supabase free tier gives you 500MB database, 1GB storage, and 50,000 users — more than enough to launch.
2
Create your ClipVault project
Takes about 2 minutes to spin up
1
New project
On your Supabase dashboard, click New project.
2
Fill in the form
Set these fields:

Name: clipvault
Database Password: Create a strong password — save it somewhere safe
Region: Pick the one closest to you (e.g. Europe West or US East)
Plan: Free
3
Wait ~2 minutes
Supabase builds your database. A progress bar will show it. When it turns green, you're ready.
3
Get your API keys
2 values — copy them into your files
1
Go to Project Settings
In your Supabase project, click the ⚙️ Settings icon in the left sidebar, then click API.
2
Copy your Project URL
Under Project URL, you'll see something like:
https://abcxyz123.supabase.co
Copy this.
3
Copy your Anon Key
Under Project API Keys, copy the anon public key. It starts with eyJ...
4
Paste both into your files
Open signup.html and login.html in a text editor (right-click → Open With → Notepad or TextEdit). Find this at the top of the <script> and replace the placeholder values:
// Replace these two lines in signup.html AND login.html
const SUPABASE_URL = 'YOUR_SUPABASE_URL';
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';

// Becomes (example):
const SUPABASE_URL = 'https://abcxyz123.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';
Never share your service_role key. Only the anon key goes into your HTML files — it's safe to expose.
4
Enable Email + Google login
Turn on auth providers — 2 minutes
1
Open Authentication settings
In Supabase left sidebar, click Authentication → then Providers.
2
Email is already on
Email/password auth is enabled by default. Leave it as-is. ✓
3
Enable Google (optional but recommended)
Click Google in the list → toggle it on.

To get Google credentials:
→ Go to console.cloud.google.com ↗
→ Create a project → APIs & Services → Credentials → Create OAuth 2.0 Client ID
→ Paste the Client ID and Secret back into Supabase

This step is optional — email signup works perfectly without it.
4
Add your site URL
In Supabase → Authentication → URL Configuration, set:
Site URL → your Vercel URL (e.g. https://clipvault.vercel.app)
Redirect URLs → same URL + /dashboard.html
Users can now create accounts with email and get a confirmation email automatically.
5
Deploy to Vercel
Your site goes live in under 5 minutes
Make sure you've pasted your Supabase URL and key into signup.html and login.html before this step.
1
Your folder should contain these files
index.html — Landing page
dashboard.html — App dashboard
signup.html — Sign up page
login.html — Log in page
2
Go to Vercel
Go to vercel.com ↗ → Sign up with GitHub (free) → click Add New → Project.
3
Deploy your folder
Click "Deploy from your computer" or drag your clipvault folder into the Vercel deploy window. No configuration needed — just drop and deploy.
4
Get your live URL
Vercel gives you a URL like https://clipvault-xyz.vercel.app. Copy it and paste it back into Supabase → Authentication → URL Configuration as your Site URL.
5
Test it live
Visit your URL → click "Start free" → sign up with an email → check your inbox for the confirmation email → log in. 🎉
Every time you update a file and re-upload to Vercel, your site updates automatically within seconds.
🎉

Auth is live!

Real users can now sign up, log in, and reset their password.
Your site is deployed and working.
